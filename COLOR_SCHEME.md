# Todoアプリ 配色設計書（改訂版）

## 🎨 配色コンセプト

**「やさしい背景から、操作部分へと徐々に締まっていく印象」**

6色を明度順に配置し、上から下（または重要度が上がる）につれて色が濃くなる設計です。

### 色の明度順（薄い→濃い）

| 順番 | カラーコード | 色名 | 役割 |
|------|------------|------|------|
| 1 | `#FAE5D8` | 背景固定 | アプリ全体の背景（固定） |
| 2 | `#FBE4D8` | カード背景 | メインカード・完了Todo |
| 3 | `#DFB6B2` | アクセント | 境界線・非アクティブボタン・カテゴリバッジ |
| 4 | `#854F6C` | サブ操作 | サブボタン・フィルター（アクティブ）・チェックボックス |
| 5 | `#522B5B` | 主要操作 | 主要ボタン（作成など） |
| 6 | `#2A114B` | 最重要操作 | ホバー・強調・フォーカス |

---

## 📐 各UI要素の色割り当て

### 1. メインカード（コンテナ）
- **背景**: 白 (`#FFFFFF`)
- **境界線**: `#DFB6B2` (アクセント)
- **理由**: コンテンツを際立たせ、背景との区別を明確に

### 2. 未完了Todoカード
- **背景**: 白 (`#FFFFFF`)
- **境界線**: `#DFB6B2` (アクセント)
- **理由**: 未完了状態を明確に示し、背景と区別

### 3. 完了Todoカード
- **背景**: `#FBE4D8` (カード背景)
- **境界線**: `#DFB6B2` (アクセント)
- **理由**: 完了状態を視覚的に区別しつつ、やさしい印象を保つ

### 4. 主要ボタン（追加・作成など）
- **通常**: `#522B5B` (主要操作)
- **ホバー**: `#2A114B` (最重要操作)
- **理由**: 最も重要な操作なので、濃い色で強調

### 5. サブボタン / フィルター
- **アクティブ**: `#854F6C` (サブ操作)
- **非アクティブ**: `#DFB6B2` (アクセント)
- **ホバー（非アクティブ時）**: `#854F6C` (サブ操作)
- **理由**: 主要ボタンより控えめに、でも操作可能であることを示す

### 6. チェック操作部分（チェックボックス）
- **色**: `#854F6C` (サブ操作)
- **理由**: サブ操作として、統一感を保つ

### 7. カテゴリバッジ
- **未完了Todo**: `#DFB6B2` (アクセント) 背景
- **完了Todo**: `#DFB6B2` (アクセント) 50%透明度
- **理由**: 情報表示として控えめに

### 8. 入力欄（フォーカス時）
- **フォーカスリング**: `#854F6C` (サブ操作)
- **境界線**: `#522B5B` (主要操作)
- **理由**: 入力可能であることを明確に示す

## 🔄 「一色ずつずらす」の具体例

### 例1: ボタンの階層

```
背景 (#FAE5D8)
  ↓
非アクティブフィルター (#DFB6B2) ← 1段階濃い
  ↓
アクティブフィルター (#854F6C) ← さらに1段階濃い
  ↓
主要ボタン (#522B5B) ← さらに1段階濃い
  ↓
主要ボタンホバー (#2A114B) ← 最も濃い
```

### 例2: Todoカードの階層

```
アプリ背景 (#FAE5D8)
  ↓
完了Todo背景 (#FBE4D8) ← 1段階濃い（ほぼ同じだが区別）
  ↓
未完了Todo背景 (白) ← より明確な区別
  ↓
境界線 (#DFB6B2) ← アクセントとして
```

### 例3: 操作要素の階層

```
カテゴリバッジ (#DFB6B2) ← 情報表示
  ↓
チェックボックス (#854F6C) ← サブ操作
  ↓
フィルターボタン（アクティブ） (#854F6C) ← サブ操作
  ↓
主要ボタン (#522B5B) ← 主要操作
  ↓
主要ボタンホバー (#2A114B) ← 最重要操作
```

**「ずらす」の意味**: 各UI要素が1色ずつ異なる明度・役割を持ち、視覚的な階層を形成すること

## 💻 CSS変数での定義例

```css
:root {
  /* 配色パレット（明度順） */
  --color-bg: #FAE5D8;        /* 1. 背景固定 */
  --color-card: #FBE4D8;      /* 2. カード背景 */
  --color-accent: #DFB6B2;    /* 3. アクセント */
  --color-sub: #854F6C;       /* 4. サブ操作 */
  --color-primary: #522B5B;   /* 5. 主要操作 */
  --color-emphasis: #2A114B;  /* 6. 最重要操作 */
  
  /* セマンティックカラー */
  --bg-main: var(--color-bg);
  --bg-card: #FFFFFF;
  --bg-card-completed: var(--color-card);
  --border-default: var(--color-accent);
  --button-primary: var(--color-primary);
  --button-primary-hover: var(--color-emphasis);
  --button-secondary: var(--color-accent);
  --button-sub: var(--color-sub);
  --accent-active: var(--color-sub);
  --accent-checkbox: var(--color-sub);
  --focus-ring: var(--color-sub);
  --focus-border: var(--color-primary);
  --text-color: #000000;
}
```

## 🎯 改善案・微調整

### 案1: より控えめな印象
- 主要ボタン: `#854F6C` → ホバー時 `#522B5B`
- サブボタン: `#DFB6B2` → ホバー時 `#854F6C`
- **効果**: より落ち着いた印象、操作の差別化が控えめ

### 案2: より明確な階層
- 主要ボタン: `#522B5B` → ホバー時 `#2A114B`（現行案）
- サブボタン: `#854F6C` → ホバー時 `#522B5B`
- **効果**: 操作の重要度がより明確

### 案3: カードの区別を強化
- 未完了Todo: 白背景 + `#FBE4D8` の薄い境界線
- 完了Todo: `#FBE4D8` 背景 + `#DFB6B2` 境界線
- **効果**: 完了/未完了の区別がより明確

---

## ✅ 実装のポイント

1. **CSS変数の活用**: 色の変更が容易
2. **一貫性**: 同じ役割の要素は同じ色を使用
3. **視覚的階層**: 明度順に配置することで、自然な視線の流れを実現
4. **操作の明確化**: 重要度に応じて色の濃さを変えることで、操作可能な場所が分かりやすい
