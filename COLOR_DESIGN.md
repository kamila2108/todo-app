# Todoアプリ 配色設計書

## 📐 配色コンセプト

**「落ち着いた大人の雰囲気を保ちながら、操作しやすいUI」**

指定された4色（#FBE4D8, #DFB6B2, #854F6C, #522B5B）を使用し、視認性と使いやすさを最優先に設計しました。

---

## 🎨 配色パレット

| 色名 | カラーコード | 用途 | CSS変数名 |
|------|------------|------|----------|
| **Peach（ピーチ）** | `#FBE4D8` | 背景・完了Todo | `--color-peach` |
| **Rose（ローズ）** | `#DFB6B2` | 境界線・非アクティブボタン・カテゴリバッジ | `--color-rose` |
| **Dusty（ダスティ）** | `#854F6C` | 主要ボタン・アクティブ状態・チェックボックス | `--color-dusty` |
| **Dark（ダーク）** | `#522B5B` | ホバー・強調 | `--color-dark` |

---

## 🎯 各要素の色配置

### 1. ページ全体の背景
- **色**: `#FBE4D8` (Peach)
- **理由**: 最も薄い色で目に優しく、長時間使用しても疲れにくい

### 2. メインカード（コンテナ）
- **背景**: 白 (`#FFFFFF`)
- **境界線**: `#DFB6B2` (Rose)
- **理由**: 白背景でコンテンツを際立たせ、Roseの境界線で優しい印象を演出

### 3. 未完了Todoカード
- **背景**: 白 (`#FFFFFF`)
- **境界線**: `#DFB6B2` (Rose)
- **理由**: 白背景で「未完了」であることを明確に示す

### 4. 完了Todoカード
- **背景**: `#FBE4D8` (Peach)
- **境界線**: `#DFB6B2` (Rose)
- **理由**: Peach背景で「完了」状態を視覚的に区別しやすくする

### 5. 追加ボタン（作成ボタン）
- **通常**: `#854F6C` (Dusty)
- **ホバー**: `#522B5B` (Dark)
- **理由**: 最も重要な操作なので、濃い色で目立たせる。ホバーでさらに濃くして操作可能であることを示す

### 6. チェックボックス
- **色**: `#854F6C` (Dusty)
- **理由**: 主要操作の一つなので、Dustyで統一感を保つ

### 7. フィルター/カテゴリ切り替えボタン
- **アクティブ**: `#854F6C` (Dusty) 背景 + 白文字
- **非アクティブ**: `#DFB6B2` (Rose) 背景 + 黒文字
- **理由**: アクティブ状態を明確に区別し、操作可能な場所が分かりやすい

### 8. 入力欄（フォーカス時）
- **フォーカスリング**: `#854F6C` (Dusty)
- **境界線**: `#854F6C` (Dusty)
- **理由**: 入力可能であることを明確に示す

### 9. カテゴリバッジ
- **未完了Todo**: `#DFB6B2` (Rose) 背景 + 黒文字
- **完了Todo**: `#DFB6B2` (Rose) 50%透明度 + 黒文字
- **理由**: カテゴリ情報を控えめに表示しつつ、視認性を確保

### 10. エラー/削除ボタン
- **色**: 赤系（既存の赤を維持）
- **理由**: 危険な操作であることを明確に示すため、配色パレット外の色を使用

---

## 💡 配色の理由（詳細）

### 視認性の確保
- **白とPeachのコントラスト**: 未完了/完了を明確に区別
- **黒文字の統一**: すべての文字色を黒に統一することで、可読性を最大化
- **Dustyの使用**: 主要操作に濃い色を使用し、クリック可能な場所を明確に

### 落ち着いた雰囲気
- **パステル系の統一**: 4色すべてがパステル系で統一され、落ち着いた印象
- **派手さの抑制**: 鮮やかな色を避け、大人っぽい雰囲気を実現

### 操作の明確化
- **アクティブ/非アクティブの区別**: DustyとRoseで明確に区別
- **ホバー効果**: Dark色でホバー状態を明確に示す
- **フォーカスリング**: 入力欄のフォーカス時にDustyのリングを表示

## 🛠️ CSS変数の使用例

```css
:root {
  /* 配色パレット */
  --color-peach: #FBE4D8;
  --color-rose: #DFB6B2;
  --color-dusty: #854F6C;
  --color-dark: #522B5B;
  
  /* セマンティックカラー */
  --bg-main: var(--color-peach);
  --bg-card: #FFFFFF;
  --bg-card-completed: var(--color-peach);
  --border-default: var(--color-rose);
  --button-primary: var(--color-dusty);
  --button-primary-hover: var(--color-dark);
  --button-secondary: var(--color-rose);
  --accent-active: var(--color-dusty);
  --text-color: #000000;
}
```

### Tailwind設定での使用

```typescript
// tailwind.config.ts
colors: {
  'todo-peach': '#FBE4D8',
  'todo-rose': '#DFB6B2',
  'todo-dusty': '#854F6C',
  'todo-dark': '#522B5B',
}
```

### 使用例

```tsx
// ボタン
<button className="bg-todo-dusty hover:bg-todo-dark text-white">
  作成
</button>

// カード
<div className="bg-white border border-todo-rose">
  {/* 未完了Todo */}
</div>
<div className="bg-todo-peach border border-todo-rose">
  {/* 完了Todo */}
</div>
```
