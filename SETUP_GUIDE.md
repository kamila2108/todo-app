# Todoアプリ - Windows環境での実行手順

このガイドでは、Windows環境でTodoアプリをローカルで実行する手順を、初心者の方にも分かりやすく説明します。

---

## 📋 前提条件

このアプリを実行するには、以下のソフトウェアが必要です：

1. **PowerShell** (Windows 10以降は標準でインストールされています)
2. **Node.js** (バージョン 18以上)
3. **npm** (Node.jsと一緒にインストールされます)

---

## 🔧 ステップ0: PowerShellを確認・インストールする

### 0-1. PowerShellがインストールされているか確認

**Windows 10以降の場合：**
- PowerShellは標準でインストールされています
- 以下の手順で開くことができます

**Windows 7/8の場合：**
- PowerShell 2.0以上がインストールされている必要があります
- インストールされていない場合は、以下の手順でインストールしてください

### 0-2. PowerShellを開いてみる

1. **Windowsキー**を押す
2. 「**PowerShell**」と入力
3. 「**Windows PowerShell**」または「**PowerShell**」をクリック

**もしPowerShellが見つからない場合：**
- Windows 7/8を使用している可能性があります
- 次の「0-3. PowerShellをインストールする（Windows 7/8の場合のみ）」に進んでください

### 0-3. PowerShellをインストールする（Windows 7/8の場合のみ）

#### 方法1: Windows Management Frameworkをインストール

1. ブラウザを開く
2. 以下のURLにアクセス：
   ```
   https://www.microsoft.com/ja-jp/download/details.aspx?id=54616
   ```
3. 「ダウンロード」ボタンをクリック
4. お使いのWindowsのバージョン（32bitまたは64bit）に合わせて適切なファイルをダウンロード
5. ダウンロードしたファイルをダブルクリックしてインストール
6. インストールが完了したら、PCを再起動

#### 方法2: Windows Updateからインストール

1. 「コントロールパネル」を開く
2. 「Windows Update」を開く
3. 利用可能な更新を確認
4. 「Windows Management Framework」の更新があればインストール

### 0-4. PowerShellのバージョンを確認

PowerShellを開いたら、以下のコマンドを入力してEnterキーを押します：

```powershell
$PSVersionTable.PSVersion
```

**💡 コマンドのコピー方法（初心者向け）：**

このガイドでは、コードブロック（灰色の枠で囲まれた部分）にコマンドが表示されます。

**コピーする部分：**
- ✅ **コピーする：** `$PSVersionTable.PSVersion` の部分だけ
- ❌ **コピーしない：** ` ```powershell` や ` ``` ` の行はコピーしないでください

**コピーの手順：**
1. `$PSVersionTable.PSVersion` の文字列をマウスで選択（ドラッグ）
2. `Ctrl + C` キーを押してコピー
3. PowerShellのウィンドウをクリック
4. `Ctrl + V` キーを押して貼り付け
5. Enterキーを押して実行

**または手動で入力：**
- PowerShellのウィンドウに直接 `$PSVersionTable.PSVersion` と入力してもOKです

**注意：** 以降のコマンドも同じ方法でコピーしてください。

**期待される結果：**
- バージョン番号が表示されます（例：`Major 5` または `Major 2`）
- PowerShell 2.0以上が表示されればOKです

**もしエラーが出た場合：**
- PowerShellが正しくインストールされていない可能性があります
- 上記のインストール手順を再度確認してください

### 0-5. 「最新のPowerShellをインストールしてください」というメッセージについて

PowerShellを開いたときに、以下のようなメッセージが表示されることがあります：

```
新機能と改善のために最新の PowerShell をインストールしてください!
https://aka.ms/PSWindows
```

**このメッセージについて：**

- **無視しても問題ありません！** このメッセージは単なるお知らせです
- PowerShell 5.1（Major 5）がインストールされていれば、このTodoアプリを実行するのに十分です
- このメッセージは、より新しいバージョン（PowerShell 7+）が利用可能であることを知らせているだけです

**メッセージを消したい場合：**

1. メッセージの部分を選択して削除するか、そのままEnterキーを押して続行してください
2. メッセージは毎回表示されますが、機能には影響しません

**最新版をインストールしたい場合（任意）：**

最新のPowerShell 7+をインストールすることもできますが、**このTodoアプリを実行するには必須ではありません**。

もし最新版をインストールしたい場合は：

1. ブラウザで以下のURLを開く：
   ```
   https://aka.ms/PSWindows
   ```
2. 「Download」または「ダウンロード」ボタンをクリック
3. ダウンロードしたファイル（例：`PowerShell-7.x.x-win-x64.msi`）をダブルクリック
4. インストールウィザードに従ってインストール
5. インストール後は「PowerShell 7」という名前で起動できます

**注意：**
- 最新版をインストールしても、従来の「Windows PowerShell」も引き続き使用できます
- どちらを使っても、このTodoアプリは正常に動作します

## 🔍 ステップ1: Node.jsがインストールされているか確認する

1. Windowsキーを押す

### 1-2. Node.jsのバージョンを確認

```powershell
node --version
```

**💡 コマンドのコピー方法（初心者向け）：**

上記のコードブロックからコマンドをコピーする場合：

1. **コピーする部分：** `node --version` の部分だけをコピーしてください
   - ❌ コピーしない：` ```powershell` や ` ``` ` の行
   - ✅ コピーする：`node --version` の部分だけ

2. **コピーの手順：**
   - `node --version` の文字列をマウスで選択（ドラッグ）
   - `Ctrl + C` キーを押してコピー
   - PowerShellのウィンドウをクリック
   - `Ctrl + V` キーを押して貼り付け
   - Enterキーを押して実行

3. **または手動で入力：**
   - PowerShellのウィンドウに直接 `node --version` と入力してもOKです

**期待される結果：**
- `v18.x.x` または `v20.x.x` などのバージョン番号が表示される

**もしエラーが出た場合：**

- Node.jsを再インストールしてください

## 📥 ステップ2: Node.jsをインストールする（必要な場合のみ）

### 2-1. Node.jsの公式サイトにアクセス

1. ブラウザを開く
2. 以下のURLにアクセス：
   ```
   https://nodejs.org/ja/
   ```

### 2-2. LTS版をダウンロード

1. ページに「推奨版」と書かれた緑色のボタン（例：`v20.x.x LTS`）をクリック
2. ダウンロードページに移動します

### 2-2-1. ダウンロード形式を選択

「Node.jsを入手」ボタンを押すと、以下の2つの選択肢が表示されます：

- **Windowsインストーラー** ← **こちらを選んでください！**
- スタンドアロンのバイナリー

**どちらを選ぶべきか：**

✅ **「Windowsインストーラー」を選んでください**

**理由：**
- インストールが簡単で、自動的に設定が行われます
- npmも一緒にインストールされます
- 初心者の方には最も簡単な方法です

❌ **「スタンドアロンのバイナリー」は選ばないでください**

**理由：**
- 手動で設定する必要があり、初心者には難しいです
- 環境変数の設定など、追加の作業が必要です

**手順：**
1. **「Windowsインストーラー」** のリンクをクリック
2. ダウンロードが開始されます（ファイル名は `node-v20.x.x-x64.msi` のようになります）

### 2-3. インストーラーを実行

1. ダウンロードしたファイル（例：`node-v20.x.x-x64.msi`）をダブルクリック
2. インストールウィザードが開きます
3. 「次へ」をクリックして進みます
4. ライセンス同意画面で「同意する」を選択して「次へ」
5. インストール先はそのままで「次へ」
6. 「次へ」を何度かクリックして、最後に「インストール」をクリック
7. インストールが完了したら「完了」をクリック

### 2-4. インストール確認

1. **PowerShellを閉じて、再度開く**（重要！）
2. 以下のコマンドを実行して確認：

```powershell
node --version
npm --version
```

両方のコマンドでバージョン番号が表示されればOKです！

## 📂 ステップ3: プロジェクトフォルダに移動する

### 3-1. プロジェクトフォルダの場所を確認

**プロジェクトフォルダは以下の場所にあります：**

```
C:\Users\tokam\Downloads\1
```

**このフォルダはどこにあるの？**

1. **エクスプローラーで確認する方法：**
   - Windowsキー + E キーを押してエクスプローラーを開く
   - 左側のサイドバーから「PC」または「このPC」をクリック
   - 「ローカルディスク (C:)」を開く
   - 「Users」フォルダを開く
   - 「tokam」フォルダを開く
   - 「Downloads」フォルダを開く
   - 「1」フォルダがプロジェクトフォルダです

2. **アドレスバーから直接開く方法：**
   - エクスプローラーを開く
   - 上部のアドレスバー（URLが表示されている場所）をクリック
   - 以下のパスを入力してEnterキーを押す：
     ```
     C:\Users\tokam\Downloads\1
     ```

3. **ファイル名で確認：**
   - このフォルダには以下のファイルが入っています：
     - `package.json`
     - `README.md`
     - `SETUP_GUIDE.md`
     - `app` フォルダ
     - `features` フォルダ
     - など

### 3-2. 現在の場所を確認（PowerShellで）

PowerShellで以下のコマンドを実行：

```powershell
pwd
```

現在のディレクトリ（フォルダ）のパスが表示されます。

### 3-3. プロジェクトフォルダに移動

プロジェクトフォルダは `C:\Users\tokam\Downloads\1` です。

以下のコマンドを実行：

```powershell
cd C:\Users\tokam\Downloads\1
```

**💡 コマンドのコピー方法：**
- `cd C:\Users\tokam\Downloads\1` の部分だけをコピーしてください
- ` ```powershell` や ` ``` ` の行はコピーしないでください

**確認方法：**
```powershell
pwd
```

`C:\Users\tokam\Downloads\1` と表示されればOKです！

**もし「フォルダが見つかりません」というエラーが出た場合：**
- エクスプローラーで `C:\Users\tokam\Downloads\1` フォルダが存在するか確認してください
- フォルダ名が異なる場合は、実際のフォルダ名に合わせてコマンドを修正してください

## 📦 ステップ4: 必要なパッケージをインストールする

### 4-1. インストールコマンドを実行

プロジェクトフォルダに移動した状態で、以下のコマンドを実行：

```powershell
npm install
```

**何をしているのか：**
- `package.json`というファイルに書かれている必要なライブラリ（React、Next.js、Tailwind CSSなど）を自動的にダウンロードしてインストールします
- 初回は数分かかることがあります

**期待される結果：**
- たくさんのログが表示されます
- 最後に `added XXX packages` のようなメッセージが表示されます
- エラーが出なければ成功です！

### 4-2. インストールが完了したか確認する方法

**💡 `added XXX packages` や `npm ERR!` が見えない場合：**

1. **スクロールして確認する**
   - PowerShellのウィンドウで、マウスのホイールを上にスクロール
   - または、ウィンドウの右側のスクロールバーを上にドラッグ
   - 最後の方（一番下）に `added XXX packages` というメッセージがあるか確認

2. **まだ実行中か確認する**
   - PowerShellのウィンドウの最後の行を見る
   - カーソル（`_` や `|`）が点滅している → まだ実行中です。完了まで待ちましょう
   - カーソルが止まっている → 完了しています

3. **完了の確認方法（推奨）：**
   
   **方法1: フォルダで確認**
   - エクスプローラーでプロジェクトフォルダ（`C:\Users\tokam\Downloads\1`）を開く
   - `node_modules` というフォルダが作成されていれば、インストールは成功しています
   - `package-lock.json` というファイルも作成されます

   **方法2: PowerShellで確認**
   ```powershell
   Test-Path node_modules
   ```
   - `True` と表示されれば、インストールは成功しています
   - `False` と表示されたら、インストールが完了していません

4. **エラーがないか確認する**
   - PowerShellのウィンドウで、`npm ERR!` という文字列を探す
   - `Ctrl + F` キーを押して検索ボックスを開く
   - `npm ERR!` と入力して検索
   - 見つからなければ、エラーはありません

5. **インストールが完了したか確認するコマンド**
   ```powershell
   npm list --depth=0
   ```
   - パッケージの一覧が表示されれば、インストールは成功しています
   - エラーが出た場合は、再度 `npm install` を実行してください

**✅ インストール成功の確認：**
- `node_modules` フォルダが存在する
- `package-lock.json` ファイルが存在する
- `npm list --depth=0` コマンドでパッケージが表示される

**❌ インストール失敗の場合：**
- `npm ERR!` というメッセージが表示されている
- `node_modules` フォルダが存在しない
- その場合は、次の「もしエラーが出た場合」を参照してください

**⚠️ 警告メッセージ（`npm warn`）について：**

インストール中に、以下のような警告メッセージが表示されることがあります：

```
npm warn deprecated inflight@1.0.6: ...
npm warn deprecated rimraf@3.0.2: ...
npm warn deprecated eslint@8.57.1: ...
3 high severity vulnerabilities
```

**これらの警告は無視しても問題ありません！**

**理由：**
- これらは「非推奨（deprecated）」や「脆弱性（vulnerabilities）」の警告です
- 開発環境での使用には問題ありません
- インストール自体は正常に完了しています（`added XXX packages` と表示されていればOK）

**もし気になる場合（任意）：**

1. **脆弱性を修正したい場合：**
   ```powershell
   npm audit fix
   ```
   このコマンドを実行すると、自動的に修正できる脆弱性を修正します

2. **npmを最新版に更新したい場合（任意）：**
   ```powershell
   npm install -g npm@latest
   ```
   ただし、このTodoアプリを実行するには必須ではありません

**重要：** `added XXX packages` と表示されていれば、インストールは成功しています。次のステップに進んでください！

**もしエラーが出た場合：**
- エラーメッセージを確認してください
- よくあるエラー：
  - `npm ERR!` で始まるメッセージ → ネットワークエラーの可能性があります。再度実行してみてください
  - 権限エラー → PowerShellを「管理者として実行」してから再度試してください

```
  - Local:        http://localhost:3000

```

**重要：**

## 🌐 ステップ6: ブラウザでアプリを開く

### 6-1. ブラウザを開く

1. お好きなブラウザ（Chrome、Edge、Firefoxなど）を開く
2. アドレスバーに以下を入力してEnterキーを押す：

```
http://localhost:3000
```

### 6-2. アプリが表示されることを確認

以下のような画面が表示されれば成功です：
- 「Todo アプリ」というタイトル
- 入力フォーム
- Todoリスト（まだ空の場合は「まだTodoがありません...」と表示）

### 6-3. ブラウザでエラーが表示される場合

**もしブラウザでエラーが表示される場合：**

1. **「接続できません」や「This site can't be reached」と表示される場合：**
   - ステップ5で `npm run dev` が実行されているか確認してください
   - PowerShellのウィンドウを確認し、サーバーが起動していることを確認してください
   - 問題4の「ブラウザで「接続できません」と表示される」を参照してください

2. **「404 Not Found」や「ページが見つかりません」と表示される場合：**
   - URLが正しいか確認してください：`http://localhost:3000`
   - `https://` ではなく `http://` を使用してください
   - ポート番号が `3000` になっているか確認してください

3. **「Internal Server Error」や「500エラー」と表示される場合：**
   - PowerShellのウィンドウでエラーメッセージを確認してください
   - サーバーを再起動してみてください（`Ctrl + C` で停止してから `npm run dev` で再起動）

4. **白い画面や何も表示されない場合：**
   - ブラウザをリロードしてください（F5キー）
   - ブラウザのキャッシュをクリアしてみてください（Ctrl + Shift + Delete）
   - 別のブラウザで試してみてください

5. **JavaScriptのエラーが表示される場合：**
   - ブラウザの開発者ツールを開いてエラーを確認してください（F12キー）
   - PowerShellのウィンドウでエラーメッセージを確認してください
   - コードにエラーがないか確認してください

## 🔧 よくある問題と解決方法

### 問題0: PowerShellが見つからない、または開けない

**原因：** PowerShellがインストールされていない、またはWindowsのバージョンが古い

**解決方法：**

1. **Windows 10以降の場合：**
   - PowerShellは標準でインストールされています
   - Windowsキーを押して「PowerShell」と検索してみてください
   - それでも見つからない場合：コマンドプロンプト（cmd）でも代用可能です

2. **Windows 7/8の場合：**
   - ステップ0の「0-3. PowerShellをインストールする」を参照してください
   - Windows Management Frameworkをインストールする必要があります

3. **代替方法：コマンドプロンプトを使用する**
   - PowerShellの代わりにコマンドプロンプト（cmd）も使用できます
   - Windowsキーを押して「cmd」と入力して開く
   - ただし、一部のコマンドが異なる場合があります

### 問題0-1: PowerShell起動時に「最新のPowerShellをインストールしてください」と表示される

**原因：** PowerShell 5.1がインストールされているが、より新しいバージョン（PowerShell 7+）が利用可能であることを知らせるメッセージ

**解決方法：**

1. **このメッセージは無視して問題ありません**
   - PowerShell 5.1（Major 5）があれば、このTodoアプリは正常に動作します
   - メッセージの部分を選択して削除するか、そのままEnterキーを押して続行してください

2. **メッセージを消したい場合：**
   - メッセージは毎回表示されますが、機能には影響しません
   - そのまま作業を続けて問題ありません

3. **最新版をインストールしたい場合（任意）：**
   - ステップ0の「0-5. 最新のPowerShellをインストールしてくださいというメッセージについて」を参照してください
   - ただし、このTodoアプリを実行するには必須ではありません

### 問題1: `node: command not found` エラー

**原因：** Node.jsがインストールされていない、またはパスが通っていない

**解決方法：**
1. Node.jsを再インストール（ステップ2を参照）
2. PowerShellを閉じて再度開く
3. それでもダメな場合：PCを再起動してみる

### 問題2: `npm install` でエラーが出る

**原因：** ネットワークエラー、権限エラー、または古いキャッシュ

**解決方法：**
1. インターネット接続を確認
2. PowerShellを「管理者として実行」してから再度試す
3. 以下のコマンドでキャッシュをクリア：

```powershell
npm cache clean --force
npm install
```

### 問題2-1: `npm install` の実行結果が見えない、`added XXX packages` や `npm ERR!` が見つからない

**原因：** 出力が多すぎて見つけにくい、またはスクロールして見逃している

**解決方法：**

1. **スクロールして確認する**
   - PowerShellのウィンドウで、マウスのホイールを上にスクロール
   - または、ウィンドウの右側のスクロールバーを上にドラッグ
   - 最後の方（一番下）に `added XXX packages` というメッセージがあるか確認

2. **検索機能を使う**
   - PowerShellのウィンドウで `Ctrl + F` キーを押す
   - `added` と入力して検索 → `added XXX packages` が見つかれば成功
   - `npm ERR!` と入力して検索 → 見つからなければエラーなし

3. **フォルダで確認する（最も確実）**
   - エクスプローラーでプロジェクトフォルダ（`C:\Users\tokam\Downloads\1`）を開く
   - `node_modules` というフォルダが作成されていれば、インストールは成功しています
   - `package-lock.json` というファイルも作成されます

4. **PowerShellで確認する**
   ```powershell
   Test-Path node_modules
   ```
   - `True` と表示されれば、インストールは成功しています
   - `False` と表示されたら、インストールが完了していません

5. **パッケージ一覧を確認する**
   ```powershell
   npm list --depth=0
   ```
   - パッケージの一覧が表示されれば、インストールは成功しています
   - エラーが出た場合は、再度 `npm install` を実行してください

**✅ インストール成功の確認：**
- `node_modules` フォルダが存在する
- `package-lock.json` ファイルが存在する
- `npm list --depth=0` コマンドでパッケージが表示される

**❌ インストール失敗の場合：**
- `npm ERR!` というメッセージが表示されている
- `node_modules` フォルダが存在しない
- その場合は、問題2の解決方法を参照してください

### 問題2-2: `npm install` で警告メッセージ（`npm warn`）や脆弱性（`vulnerabilities`）が表示される

**原因：** 一部のパッケージが非推奨（deprecated）になっている、または脆弱性が報告されている

**解決方法：**

1. **これらの警告は無視しても問題ありません**
   - `added XXX packages` と表示されていれば、インストールは成功しています
   - 開発環境での使用には問題ありません
   - 次のステップ（ステップ5）に進んでください

2. **警告の例：**
   ```
   npm warn deprecated inflight@1.0.6: ...
   npm warn deprecated eslint@8.57.1: ...
   3 high severity vulnerabilities
   ```
   これらは警告であり、エラーではありません

3. **もし気になる場合（任意）：**
   - 脆弱性を修正したい場合：
     ```powershell
     npm audit fix
     ```
   - npmを最新版に更新したい場合（任意）：
     ```powershell
     npm install -g npm@latest
     ```
   ただし、このTodoアプリを実行するには必須ではありません

**重要：** `added XXX packages` と表示されていれば、インストールは成功しています！

### 問題3: `port 3000 is already in use` エラー

**原因：** ポート3000が既に使用されている

**解決方法：**
1. 別のターミナルで既にサーバーが動いていないか確認
2. 動いている場合は `Ctrl + C` で停止
3. それでもダメな場合：別のポートを使用（`npm run dev -- -p 3001`）

### 問題4: ブラウザで「接続できません」やエラーが表示される

**原因：** サーバーが起動していない、URLが間違っている、またはアプリに問題がある

**解決方法：**

1. **「接続できません」や「localhost 接続が拒否されました」と表示される場合：**
   
   **これは最も一般的な問題です。サーバーが起動していない可能性が高いです。**

   **確認手順：**
   - PowerShellで `npm run dev` を実行したウィンドウが開いているか確認してください
   - ウィンドウが閉じている、または `npm run dev` を実行していない場合 → ステップ5に戻ってサーバーを起動してください

   **サーバーを再起動する方法：**
   1. PowerShellのウィンドウを開く
   2. プロジェクトフォルダに移動：
      ```powershell
      cd C:\Users\tokam\Downloads\1
      ```
   3. 開発サーバーを起動：
      ```powershell
      npm run dev
      ```
   4. `✓ Ready` や `- Local: http://localhost:3000` と表示されるまで待つ（数秒〜数十秒かかることがあります）
   5. ブラウザで `http://localhost:3000` を再度開く

   **もしエラーが表示されている場合：**
   - PowerShellのウィンドウでエラーメッセージを確認してください
   - よくあるエラー：
     - `Error: listen EADDRINUSE: address already in use` → ポート3000が既に使用されています（問題3を参照）
     - `SyntaxError` や `Module not found` → コードにエラーがあります。エラーメッセージを確認してください
   - エラーメッセージが表示されている場合は、その内容を確認して対処してください

   **その他の確認事項：**
   - URLが `http://localhost:3000` になっているか確認（`https://` ではなく `http://` です）
   - ポート番号が `3000` になっているか確認
   - ブラウザのキャッシュをクリアしてみる（Ctrl + Shift + Delete）

2. **「404 Not Found」や「ページが見つかりません」と表示される場合：**
   - URLが正しいか確認：`http://localhost:3000`
   - ポート番号が `3000` になっているか確認
   - サーバーを再起動してみてください

3. **「Internal Server Error」や「500エラー」と表示される場合：**
   - PowerShellのウィンドウでエラーメッセージを確認してください
   - サーバーを再起動してください：
     - PowerShellで `Ctrl + C` を押して停止
     - 再度 `npm run dev` を実行
   - コードにエラーがないか確認してください

4. **白い画面や何も表示されない場合：**
   - ブラウザをリロードしてください（F5キー）
   - ブラウザのキャッシュをクリアしてみてください
   - 別のブラウザ（Chrome、Edge、Firefoxなど）で試してみてください

5. **JavaScriptのエラーが表示される場合：**
   - ブラウザの開発者ツールを開いてエラーを確認（F12キー）
   - Consoleタブでエラーメッセージを確認
   - PowerShellのウィンドウでもエラーが出ていないか確認

### 問題5: コードを変更してもブラウザが更新されない

**原因：** ホットリロードが機能していない

**解決方法：**
1. ブラウザをリロード（F5キー）
2. サーバーを再起動（`Ctrl + C` で停止してから `npm run dev` で再起動）
