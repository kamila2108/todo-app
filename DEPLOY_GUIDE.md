# Todoアプリ - インターネット公開（デプロイ）ガイド

このガイドでは、作成したTodoアプリをインターネット上に公開する手順を、初心者の方にも分かりやすく説明します。

---

## 📋 このガイドで学ぶこと

このガイドを完了すると、以下のことができるようになります：

1. ✅ **GitHub**にコードをアップロードする
2. ✅ **Vercel**を使ってアプリをインターネット上に公開する
3. ✅ 世界中の誰でもアクセスできるURLを取得する

---

## 🎯 デプロイとは何か？

**デプロイ（Deploy）**とは、作成したアプリをインターネット上に公開することです。

### なぜデプロイが必要なのか？

- **現在の状態**: あなたのPC上でしか動かない（`localhost:3000`）
- **デプロイ後**: インターネット上の誰でもアクセスできるURLができる
- **メリット**: 
  - 友達や家族にアプリを見せられる
  - スマートフォンからもアクセスできる
  - 24時間いつでもアクセス可能

### このガイドで使用するサービス

1. **GitHub（ギットハブ）**
   - **何をするのか**: コードを保存・管理する場所
   - **なぜ必要か**: Vercelがコードを取得するために必要

2. **Vercel（バーセル）**
   - **何をするのか**: アプリをインターネット上に公開するサービス
   - **なぜ選ぶか**: 
     - Next.jsの開発元が提供している（最も簡単）
     - 無料プランがある
     - 自動的にデプロイしてくれる

---

## 📝 事前準備チェックリスト

デプロイを始める前に、以下が完了していることを確認してください：

- [ ] Todoアプリがローカルで正常に動作する（`npm run dev`で起動できる）
- [ ] GitHubアカウントを持っている（持っていない場合は後で作成します）
- [ ] インターネット接続がある

## 🚀 ステップ1: GitHubアカウントを作成する

### 1-1. GitHubとは何か？

**GitHub**は、プログラマーがコードを保存・共有するためのサービスです。
- **無料で使える**
- **世界中の多くの開発者が使用している**
- **コードのバックアップにもなる**

### 1-2. GitHubアカウントを作成する

1. **ブラウザを開く**
2. **以下のURLにアクセス**：
   ```
   https://github.com/signup
   ```
3. **アカウント情報を入力**：
   - **Username（ユーザー名）**: 好きな名前を入力（例: `tokam-todo`）
   - **Email（メールアドレス）**: あなたのメールアドレスを入力
   - **Password（パスワード）**: 安全なパスワードを入力
4. **「Verify your account」**をクリック
   - メールアドレスに確認コードが送られてきます
   - メールを確認して、コードを入力してください
5. **「Create account」**をクリック
6. **プランを選択**：
   - **「Free」**を選択（無料プラン）
7. **質問に答える**（任意）：
   - 興味のある分野を選択するか、「Skip」をクリック
8. **アカウント作成完了！**

**💡 コツ**: ユーザー名は後で変更できるので、気軽に決めてください。

## 🔧 ステップ2: Gitをインストールする（必要な場合）

### 2-1. Gitとは何か？

**Git（ギット）**は、コードの変更履歴を管理するツールです。
- **GitHubにコードをアップロードするために必要**
- **Windowsに標準ではインストールされていない**

### 2-2. Gitがインストールされているか確認する

1. **PowerShellを開く**
   - Windowsキーを押す
   - 「PowerShell」と入力
   - 「Windows PowerShell」をクリック

2. **以下のコマンドを実行**：
   ```powershell
   git --version
   ```

3. **結果を確認**：
   - **バージョン番号が表示される場合**（例: `git version 2.42.0`）:
     - ✅ Gitは既にインストールされています
     - **ステップ2-3はスキップ**して、ステップ3に進んでください
   - **「コマンドが見つかりません」と表示される場合**:
     - ❌ Gitがインストールされていません
     - **ステップ2-3に進んでください**

### 2-3. Gitをインストールする

1. **ブラウザを開く**
2. **以下のURLにアクセス**：
   ```
   https://git-scm.com/download/win
   ```
3. **自動的にダウンロードが始まります**
   - もし始まらない場合は、「Download for Windows」ボタンをクリック
4. **ダウンロードしたファイルを実行**
   - ファイル名は `Git-2.xx.x-64-bit.exe` のような形式です
5. **インストーラーを実行**：
   - **「Next」**をクリック（複数回）
   - **「Install」**をクリック
   - インストールが完了するまで待つ（数分かかります）
   - **「Finish」**をクリック
6. **PowerShellを再起動する**：
   - 現在開いているPowerShellを閉じる
   - 新しいPowerShellを開く
7. **インストール確認**：
   ```powershell
   git --version
   ```
   - バージョン番号が表示されれば成功です！

## 📦 ステップ3: プロジェクトフォルダをGitリポジトリにする

### 3-1. Gitリポジトリとは何か？

**Gitリポジトリ**は、Gitで管理されるフォルダのことです。
- **コードの変更履歴を記録できる**
- **GitHubにアップロードできる状態になる**

### 3-2. プロジェクトフォルダに移動する

1. **PowerShellを開く**
2. **以下のコマンドを実行**（プロジェクトフォルダのパスに置き換えてください）：
   ```powershell
   cd C:\Users\tokam\Downloads\1
   ```

**💡 プロジェクトフォルダの場所が分からない場合**：
- エクスプローラーでプロジェクトフォルダを開く
- アドレスバーをクリックして、パスをコピー
- PowerShellで `cd ` の後にペースト

### 3-3. Gitリポジトリを初期化する

**何をするのか**: このフォルダをGitで管理できるようにする

**なぜ必要か**: GitHubにアップロードするために必要

**コマンドを実行**：
```powershell
git init
```

**期待される結果**:
```
Initialized empty Git repository in C:/Users/tokam/Downloads/1/.git/
```

### 3-4. Gitの設定を行う（初回のみ）

**何をするのか**: Gitにあなたの名前とメールアドレスを登録する

**なぜ必要か**: コードの変更履歴に記録されるため

**コマンドを実行**（GitHubアカウントのメールアドレスとユーザー名に置き換えてください）：
```powershell
git config --global user.email "your-email@example.com"
git config --global user.name "Your Name"
```

**例**：
```powershell
git config --global user.email "tokam@example.com"
git config --global user.name "tokam"
```

**💡 コツ**: 
- `user.email`にはGitHubアカウントで登録したメールアドレスを入力
- `user.name`にはGitHubのユーザー名を入力（本名でもOK）

### 3-5. すべてのファイルをGitに追加する

**何をするのか**: プロジェクト内のすべてのファイルをGitの管理対象にする

**なぜ必要か**: GitHubにアップロードするファイルを指定するため

**コマンドを実行**：
```powershell
git add .
```

**期待される結果**: 何も表示されないのが正常です（エラーが出なければ成功）

### 3-6. 最初のコミットを作成する

**何をするのか**: 現在の状態を「スナップショット」として保存する

**なぜ必要か**: GitHubにアップロードする前に、変更を記録する必要があるため

**コマンドを実行**：
```powershell
git commit -m "Initial commit"
```

**期待される結果**:
```
[main (or master) xxxxxxx] Initial commit
 X files changed, Y insertions(+)
```

**💡 コツ**: `-m "Initial commit"`の部分は、この変更の説明（コミットメッセージ）です。日本語でもOKです。

## 🌐 ステップ4: GitHubにリポジトリを作成する

### 4-1. GitHubでリポジトリを作成する

**何をするのか**: GitHub上に、コードを保存する場所（リポジトリ）を作成する

**なぜ必要か**: コードをインターネット上に保存するため

1. **ブラウザでGitHubにログイン**
   - https://github.com にアクセス
   - 右上の「Sign in」をクリック
   - アカウント情報を入力してログイン

2. **新しいリポジトリを作成**：
   - 右上の「**+**」ボタンをクリック
   - 「**New repository**」をクリック

3. **リポジトリ情報を入力**：
   - **Repository name（リポジトリ名）**: `todo-app` と入力（好きな名前でOK）
   - **Description（説明）**: 「Todoアプリ」など、任意の説明を入力
   - **Public / Private**: **「Public」**を選択（無料で使えます）
   - **⚠️ 重要**: 「Add a README file」「Add .gitignore」「Choose a license」の**チェックを外す**
     - 既にプロジェクトにファイルがあるため、これらは不要です

4. **「Create repository」**をクリック

5. **次の画面が表示されます**：
   - 「Quick setup」という画面が表示されます
   - **この画面は後で使うので、開いたままにしておいてください**

## 📤 ステップ5: コードをGitHubにアップロードする

### 5-1. GitHubのURLを確認する

**何をするのか**: GitHubリポジトリのURLを確認する

**なぜ必要か**: コードをアップロードする先を指定するため

1. **GitHubの「Quick setup」画面を確認**
   - 画面に表示されているURLをコピーしてください
   - 形式は以下のようになります：
     ```
     https://github.com/your-username/todo-app.git
     ```
   - **このURLをメモ帳などに保存しておいてください**

### 5-2. リモートリポジトリを追加する

**何をするのか**: ローカルのGitリポジトリとGitHubのリポジトリを接続する

**なぜ必要か**: コードをアップロードするために必要

**コマンドを実行**（`your-username`と`todo-app`をあなたの情報に置き換えてください）：
```powershell
git remote add origin https://github.com/your-username/todo-app.git
```

**例**：
```powershell
git remote add origin https://github.com/tokam/todo-app.git
```

**期待される結果**: 何も表示されないのが正常です（エラーが出なければ成功）

### 5-3. ブランチ名を確認・変更する（必要な場合）

**何をするのか**: Gitのブランチ名を確認する

**なぜ必要か**: GitHubのデフォルトブランチ名と一致させるため

**コマンドを実行**：
```powershell
git branch
```

**結果を確認**：
- **`* main`** と表示される場合: ✅ そのまま進んでください
- **`* master`** と表示される場合: 以下のコマンドで`main`に変更してください：
  ```powershell
  git branch -M main
  ```

### 5-4. コードをGitHubにアップロードする

**何をするのか**: ローカルのコードをGitHubに送信する

**なぜ必要か**: インターネット上にコードを保存するため

**コマンドを実行**：
```powershell
git push -u origin main
```

**初回の場合、認証が求められます**：

1. **ユーザー名を入力**：
   - GitHubのユーザー名を入力してEnter

2. **パスワードを入力**：
   - **⚠️ 重要**: ここで入力するのは、GitHubのパスワード**ではなく**、**Personal Access Token（個人アクセストークン）**です
   - パスワードを入力する方法は、次の「5-5. Personal Access Tokenを作成する」を参照してください

**期待される結果**:
```
Enumerating objects: XX, done.
Counting objects: 100% (XX/XX), done.
Writing objects: 100% (XX/XX), done.
To https://github.com/your-username/todo-app.git
 * [new branch]      main -> main
Branch 'main' set up to track 'remote branch 'main' from 'origin'.
```

### 5-5. Personal Access Tokenを作成する（必要な場合のみ）

**⚠️ 注意**: もし`git push`コマンドでブラウザ認証が成功した場合（「Authorize git-ecosystem」をクリックした場合）、この手順は**スキップ**して「5-5-1. アップロードが成功したか確認する」に進んでください。

**何をするのか**: GitHubにコードをアップロードするための認証トークンを作成する

**なぜ必要か**: セキュリティのため、GitHubはパスワードの代わりにトークンを使用します（ブラウザ認証が使えない場合のみ）

#### 方法1: Settingsページから（推奨）

1. **GitHubにログイン**
2. **右上のアイコンをクリック** → 「**Settings**」をクリック
3. **左側のメニューを下にスクロール**して、「**Developer settings**」を探す
   - もし見つからない場合は、**方法2**を試してください
4. **「Personal access tokens」** → **「Tokens (classic)」**をクリック
5. **「Generate new token」** → **「Generate new token (classic)」**をクリック
6. **トークン情報を入力**：
   - **Note（メモ）**: 「Todo App Deploy」など、任意の名前を入力
   - **Expiration（有効期限）**: 「90 days」または「No expiration」を選択
   - **Select scopes（権限）**: **「repo」**にチェックを入れる
7. **「Generate token」**をクリック
8. **⚠️ 重要**: 表示されたトークンを**すぐにコピー**してください
   - 例: `ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`
   - **この画面を閉じると、もう一度確認できません**
   - メモ帳などに保存しておいてください

#### 方法2: 直接URLでアクセス

1. **ブラウザで以下のURLに直接アクセス**：
   ```
   https://github.com/settings/tokens
   ```
2. **「Generate new token」** → **「Generate new token (classic)」**をクリック
3. **トークン情報を入力**（方法1の手順6と同じ）
4. **「Generate token」**をクリック
5. **トークンをコピー**して保存

#### 使用方法

PowerShellで`git push`コマンドを実行したときにパスワードを求められたら：
- **ユーザー名**: GitHubのユーザー名を入力
- **パスワード**: コピーしたトークンを貼り付け（表示されませんが、入力されています）

**💡 コツ**: トークンは後で使うので、安全な場所に保存しておいてください。

### 5-5-1. アップロードが成功したか確認する

1. **ブラウザでGitHubのリポジトリページを開く**
   - https://github.com/your-username/todo-app にアクセス
   - `your-username`と`todo-app`をあなたの情報に置き換えてください

2. **ファイルが表示されているか確認**：
   - `app`フォルダ
   - `components`フォルダ
   - `package.json`
   - など、プロジェクトのファイルが表示されていれば成功です！

## 🚀 ステップ6: Vercelでデプロイする

### 6-1. Vercelアカウントを作成する

**何をするのか**: Vercelのアカウントを作成する

**なぜ必要か**: アプリをインターネット上に公開するために必要

#### 手順1: Vercelのサインアップページにアクセス

1. **ブラウザを開く**
2. **以下のURLに直接アクセス**：
   ```
   https://vercel.com/signup
   ```
   - ⚠️ **重要**: GitHubから飛ばされたページではなく、このURLに直接アクセスしてください

#### 手順2: GitHubアカウントでサインアップ

3. **「Continue with GitHub」**ボタンをクリック
   - GitHubアカウントでサインアップするのが最も簡単です
   - 他の方法（Email、GitLab、Bitbucket）もありますが、GitHubが推奨です

4. **GitHubの認証画面が表示される**：
   - 「Authorize Vercel」をクリック
   - 必要に応じて、GitHubのパスワードを入力
   - 2要素認証を設定している場合は、認証コードを入力

5. **Vercelのアカウント作成完了**：
   - 自動的にVercelのダッシュボードに移動します
   - 初回の場合、「Welcome to Vercel」という画面が表示されることがあります

#### 手順3: 初期設定をスキップ（推奨）

6. **Vercelの設定画面が表示される場合**：
   - **「Skip」**または「Skip this step」をクリック（後で設定できます）
   - チーム名やプロジェクト名の設定は後で変更できます

#### ⚠️ 問題が発生した場合

**「アカウントがありません」と表示される場合**：
- **原因**: Vercelのログインページ（`https://vercel.com/login`）にアクセスしてしまっている可能性があります
- **解決方法**: 
  1. ブラウザのアドレスバーに `https://vercel.com/signup` と入力してアクセス
  2. 「Continue with GitHub」をクリック
  3. GitHubで認証すると、自動的にVercelアカウントが作成されます

**「Continue with GitHub」ボタンが見つからない場合**：
- ページを下にスクロールして探してください
- または、ページをリロード（F5キー）してみてください

### 6-2. プロジェクトをインポートする

**何をするのか**: GitHubのリポジトリをVercelに接続する

**なぜ必要か**: Vercelがコードを取得してデプロイするため

#### 手順1: プロジェクト追加ボタンを見つける

Vercelのダッシュボードで、以下のいずれかの方法でプロジェクトを追加できます：

**方法1: ダッシュボードの中央**
- 画面の中央に「**Add New Project**」または「**New Project**」という大きなボタンが表示されている場合、それをクリック

**方法2: 左上のボタン**
- 画面の左上に「**+ New Project**」または「**Add New...**」というボタンがある場合、それをクリック
- ドロップダウンメニューが表示されたら、「**Project**」をクリック

**方法3: 直接URLでアクセス**
- ブラウザのアドレスバーに以下を入力：
  ```
  https://vercel.com/new
  ```
- このURLに直接アクセスすると、プロジェクト追加画面が開きます

#### 手順2: GitHubリポジトリを選択

2. **「Import Git Repository」画面が表示されます**
   - GitHubのリポジトリ一覧が表示されます
   - **作成した`todo-app`リポジトリを探してクリック**
   - リポジトリ名で検索できる場合もあります

3. **「Install the GitHub application」というメッセージが表示される場合**：
   - ⚠️ **これは必要です！** VercelがGitHubからコードを取得するために、GitHubアプリのインストール（権限の付与）が必要です
   - **「Install the GitHub application」**または「**Configure GitHub App**」ボタンをクリック
   - GitHubの認証画面が表示されます
   - **「Authorize Vercel」**をクリック
   - 必要に応じて、GitHubのパスワードを入力
   - 2要素認証を設定している場合は、認証コードを入力
   - 権限を付与すると、リポジトリ一覧が表示されるようになります

4. **リポジトリが見つからない場合**：
   - ページをリロード（F5キー）してみてください
   - または、GitHubアカウント名で検索してみてください

#### 手順3: GitHubリポジトリを選択

4. **リポジトリ一覧から`todo-app`を選択**：
   - 作成した`todo-app`リポジトリを探してクリック
   - リポジトリ名で検索できる場合もあります

#### 手順4: プロジェクト設定を確認

5. **プロジェクト設定画面が表示されます**
   - 以下の設定を確認してください（通常は自動的に正しく設定されています）：

   - **Project Name（プロジェクト名）**: `todo-app` のまま（変更可能）
   - **Framework Preset**: 「**Next.js**」が自動選択されているはずです
   - **Root Directory**: 「**`./`**」のまま（変更不要）
   - **Build Command**: 「**`npm run build`**」のまま（変更不要）
   - **Output Directory**: 「**next.js default**」または「**.next**」と表示されている場合は**そのまま**（変更不要）
     - ⚠️ **注意**: 空欄ではなく「next.js default」と表示されている場合でも、**そのままで問題ありません**
     - Next.jsのデフォルトの出力ディレクトリ（`.next`）が自動的に設定されています
   - **Install Command**: 「**`npm install`**」のまま（変更不要）

6. **設定を確認したら、「**Deploy**」ボタンをクリック**
   - 画面の下部または右上に「Deploy」ボタンがあります

### 6-3. デプロイの進行を確認する

**何をするのか**: デプロイの進行状況を確認する

**なぜ必要か**: エラーがないか確認するため

1. **デプロイ画面が表示されます**：
   - 「Building」→「Deploying」→「Ready」と進行します
   - **通常、2〜5分程度かかります**

2. **「inspect deployment」ボタンが表示された場合**：
   - ⚠️ **これは正常です！** デプロイの詳細を確認できるボタンです
   - このボタンをクリックすると、デプロイの進行状況やログを詳しく見ることができます
   - **クリックしなくてもデプロイは続行されます**

3. **進行状況を確認**：
   - 画面に表示されるログを確認してください
   - 「Building...」「Deploying...」などのメッセージが表示されます
   - エラーメッセージが表示されていないか確認してください

4. **⚠️ 黄色い警告マーク（△！）と `npm warn deprecated` メッセージが表示される場合**：
   - **これは問題ありません！** 警告（warning）であり、エラーではありません
   - 以下のようなメッセージが表示されることがあります：
     ```
     npm warn deprecated rimraf@3.0.2: Rimraf versions prior to v4 are no longer supported
     npm warn deprecated eslint@8.57.1: This version is no longer supported
     ```
   - **意味**: 使用しているパッケージの一部が古いバージョンであることを示しています
   - **影響**: アプリの動作には**全く影響しません**
   - **対処**: **何もする必要はありません**。そのままデプロイを続行してください
   - デプロイは正常に進行しています
   - ⚠️ **重要**: 黄色い警告マーク（△！）が表示されていても、デプロイは**正常に進行しています**
     - エラー（赤い×マーク）ではないので、そのまま待ってください
     - 「Building」や「Deploying」と表示されていれば、デプロイは続行されています

4. **デプロイ完了の確認**：
   - 「Build Completed」または「Ready」と表示されれば成功です！
   - 「Congratulations!」という画面が表示される場合もあります
   - **「Visit」**ボタンが表示されたら、それをクリックすると公開されたアプリが開きます

5. **デプロイが完了するまで待つ**：
   - 画面の上部に進行状況が表示されます
   - 「Building」→「Deploying」→「Ready」と変化します
   - **完了するまで数分かかりますので、そのまま待ってください**

6. **⚠️ 「Deployment failed with error」と表示される場合**：
   - **エラーの詳細を確認する必要があります**
   - 以下の手順でエラーログを確認してください

#### エラーログの確認方法

1. **「inspect deployment」ボタンをクリック**
   - デプロイの詳細ページが開きます

2. **「Build Logs」または「Logs」タブをクリック**
   - エラーメッセージが表示されます

3. **エラーメッセージを確認**
   - 赤いエラーメッセージを探してください
   - よくあるエラーと解決方法は、次のセクションを参照してください

#### よくあるエラーと解決方法

**エラー1: `Module not found` または `Cannot find module`**
- **原因**: 必要なパッケージがインストールされていない、またはインポートパスが間違っている
- **解決方法**: 
  1. ローカルで `npm install` を実行して、すべての依存関係がインストールされているか確認
  2. `package.json` に必要なパッケージが含まれているか確認
  3. コードのインポート文が正しいか確認

**エラー2: `TypeScript error` または `Type error`**
- **原因**: TypeScriptの型エラー
- **解決方法**:
  1. ローカルで `npm run build` を実行してエラーを確認
  2. TypeScriptのエラーを修正
  3. 修正後、再度コミットしてプッシュ

**エラー3: `Build failed` または `Command failed`**
- **原因**: ビルドコマンドが失敗している
- **解決方法**:
  1. ローカルで `npm run build` を実行して、同じエラーが発生するか確認
  2. エラーメッセージに従って修正
  3. 修正後、再度コミットしてプッシュ

**エラー4: `Environment variable` または `env` 関連のエラー**
- **原因**: 必要な環境変数が設定されていない
- **解決方法**: このTodoアプリでは環境変数は不要なので、このエラーは通常発生しません

**エラー5: `Node.js version` 関連のエラー**
- **原因**: Node.jsのバージョンが合わない
- **解決方法**: Vercelの設定でNode.jsのバージョンを指定（通常は自動検出されます）

#### エラーを修正した後の再デプロイ

**何をするのか**: 修正したコードをGitHubにアップロードして、Vercelが自動的に再デプロイするようにする

**なぜ必要か**: コードを修正しても、GitHubにアップロードしないとVercelは新しいコードを取得できません

##### 手順1: PowerShellを開く

1. **Windowsキー**を押す
2. 「**PowerShell**」と入力
3. 「**Windows PowerShell**」をクリック

##### 手順2: プロジェクトフォルダに移動する

**何をするのか**: PowerShellでプロジェクトフォルダに移動する

**なぜ必要か**: Gitコマンドを実行するには、プロジェクトフォルダにいる必要があります

**コマンドを実行**（プロジェクトフォルダのパスに置き換えてください）：
```powershell
cd C:\Users\tokam\Downloads\1
```

**期待される結果**: 
- 何も表示されないか、エラーメッセージが表示されない場合は成功です
- PowerShellのプロンプト（`PS C:\Users\tokam\Downloads\1>`）が表示されます

**💡 プロジェクトフォルダの場所が分からない場合**：
- エクスプローラーでプロジェクトフォルダを開く
- アドレスバーをクリックして、パスをコピー
- PowerShellで `cd ` の後にペースト

##### 手順3: 変更したファイルをGitに追加する

**何をするのか**: 修正したファイルをGitの管理対象に追加する

**なぜ必要か**: Gitに「このファイルを変更しました」と伝えるため

**コマンドを実行**：
```powershell
git add .
```

**期待される結果**: 
- 何も表示されないのが正常です（エラーが出なければ成功）
- 警告メッセージ（`warning: LF will be replaced by CRLF`）が表示される場合がありますが、**無視して大丈夫です**

**💡 コマンドの意味**：
- `git add` = Gitに「このファイルを追加/変更しました」と伝える
- `.` = 現在のフォルダ内のすべてのファイルを意味する

##### 手順4: 変更をコミットする

**何をするのか**: 変更内容を「スナップショット」として保存する

**なぜ必要か**: GitHubにアップロードする前に、変更を記録する必要があるため

**コマンドを実行**：
```powershell
git commit -m "Fix Todo type inconsistency"
```

**期待される結果**:
```
[main xxxxxxx] Fix Todo type inconsistency
 X files changed, Y insertions(+), Z deletions(-)
```

**💡 コマンドの意味**：
- `git commit` = 変更を「スナップショット」として保存する
- `-m "Fix Todo type inconsistency"` = この変更の説明（コミットメッセージ）を指定する
- メッセージは日本語でもOKです（例: `"Todo型の不一致を修正"`）

**⚠️ もし「Please tell me who you are」というエラーが出た場合**：
- Gitの設定がまだ完了していない可能性があります
- 以下のコマンドを実行してください（GitHubアカウントの情報に置き換えてください）：
  ```powershell
  git config --global user.email "your-email@example.com"
  git config --global user.name "Your Name"
  ```
- その後、再度 `git commit` を実行してください

##### 手順5: GitHubにプッシュする

**何をするのか**: ローカルの変更をGitHubにアップロードする

**なぜ必要か**: VercelがGitHubからコードを取得してデプロイするため

**コマンドを実行**：
```powershell
git push
```

**期待される結果**:
```
Enumerating objects: X, done.
Counting objects: 100% (X/X), done.
Delta compression using up to 8 threads
Compressing objects: 100% (X/X), done.
Writing objects: 100% (X/X), done.
To https://github.com/kamila2108/todo-app.git
   xxxxxxx..yyyyyyy  main -> main
```

**💡 コマンドの意味**：
- `git push` = ローカルの変更をGitHubにアップロードする

**⚠️ もし認証が求められた場合**：
- ブラウザが自動的に開いて、GitHubの認証画面が表示されます
- 「Authorize git-ecosystem」をクリックしてください
- 認証が完了すると、自動的にプッシュが続行されます

##### 手順6: Vercelで再デプロイを確認する

**何をするのか**: Vercelのダッシュボードで、自動的に再デプロイが開始されたか確認する

**なぜ必要か**: デプロイが成功したか確認するため

1. **ブラウザでVercelのダッシュボードを開く**
   - https://vercel.com/dashboard にアクセス
   - または、既に開いているVercelのページをリロード（F5キー）

2. **プロジェクトを選択**
   - `todo-app` プロジェクトをクリック

3. **デプロイの進行状況を確認**
   - 「Deployments」タブをクリック
   - 一番上に新しいデプロイが表示されます
   - 「Building」→「Deploying」→「Ready」と進行します
   - **通常、2〜5分程度かかります**

4. **デプロイが完了したら**
   - 「Ready」と表示されれば成功です！
   - 「Visit」ボタンをクリックして、アプリが正常に動作するか確認してください

**💡 コマンドのコピー方法（初心者向け）**：
- 上記のコードブロック（```powershell と ``` の間）から、**コマンドの部分だけ**をコピーしてください
- 例: `git add .` という行をコピー
- PowerShellに貼り付けてEnterキーを押してください

## ❓ よくある質問（FAQ）

### Q1: デプロイに費用はかかりますか？

**A**: Vercelの無料プランで十分です。以下の制限がありますが、個人のプロジェクトには問題ありません：
- 月100GBの帯域幅
- 無制限のデプロイ
- 無制限のプロジェクト

### Q2: デプロイに時間はかかりますか？

**A**: 初回デプロイは2〜5分程度です。2回目以降は1〜3分程度です。

### Q3: デプロイしたアプリを削除したい場合は？

**A**: Vercelのダッシュボードでプロジェクトを選択し、「Settings」→「Delete Project」から削除できます。

### Q4: エラーが発生した場合は？

**A**: 以下の点を確認してください：
1. **GitHubにコードが正しくアップロードされているか**
2. **Vercelのデプロイログにエラーがないか**
3. **`package.json`に`build`スクリプトがあるか**

### Q5: カスタムドメイン（独自のURL）を使いたい場合は？

**A**: Vercelの無料プランでもカスタムドメインを使用できます：
1. Vercelのダッシュボードでプロジェクトを選択
2. 「Settings」→「Domains」を開く
3. ドメイン名を入力して追加
