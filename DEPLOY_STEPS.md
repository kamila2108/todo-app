# 変更をデプロイする手順

## 現在の変更をVercel（ホームページ）に反映させる手順

以下の手順で、ローカルの変更をインターネット上に公開します。

**💡 Supabase対応の場合：**
Supabaseを使ったログイン対応アプリをデプロイする場合は、**`DEPLOY_SUPABASE.md`** を参照してください。
特に、**環境変数の設定（ステップ2）**が必要です。

---

## ステップ1: プロジェクトフォルダを開く

1. エクスプローラーで `C:\Users\tokam\Downloads\1` を開く
2. フォルダ内で右クリック → 「ターミナルで開く」または「PowerShellで開く」

---

## ステップ2: 変更を確認する

PowerShellで以下のコマンドを実行：

```powershell
git status
```

**期待される結果：**
- 変更されたファイルの一覧が表示されます
- 例：`modified: app/globals.css`、`modified: features/todo/components/TodoApp.tsx` など

---

## ステップ3: 変更をステージングする

すべての変更をステージング（コミットの準備）します：

```powershell
git add .
```

**何をしているのか：**
- 変更されたファイルをGitに「この変更をコミットします」と伝えます
- `.` は「すべてのファイル」を意味します

---

## ステップ4: 変更をコミットする

変更にメッセージを付けてコミットします：

```powershell
git commit -m "Update layout: 2-column design with 30-70 ratio and color scheme"
```

**何をしているのか：**
- 変更を「コミット」という形で記録します
- `-m` の後の文字列は、変更内容を説明するメッセージです
- 日本語や英語、どちらでもOKです

**別のメッセージ例：**
```powershell
git commit -m "配色とレイアウトの更新：2カラムデザイン実装"
```

---

## ステップ5: GitHubにプッシュする

変更をGitHubにアップロードします：

```powershell
git push
```

**何をしているのか：**
- ローカルの変更をGitHubに送信します
- Vercelが自動的に変更を検知して、デプロイを開始します

**もしエラーが出た場合：**
- 認証画面が表示されたら、「Authorize git-ecosystem」をクリック
- または、以前と同じ認証手順を実行してください

---

## ステップ6: Vercelでデプロイ状況を確認する

1. ブラウザで [Vercelのダッシュボード](https://vercel.com/dashboard) を開く
2. 自分のプロジェクトをクリック
3. 「Deployments」タブを確認
4. 新しいデプロイが進行中または完了していることを確認

**デプロイが完了するまでの時間：**
- 通常、2〜5分程度かかります
- 「Ready」または「Build Completed」と表示されれば完了です

---

## ステップ7: 公開サイトを確認する

デプロイが完了したら：

1. Vercelのダッシュボードで「Visit」ボタンをクリック
2. または、以前設定したURL（例：`https://your-project.vercel.app`）を開く
3. 変更が反映されていることを確認

**確認ポイント：**
- ✅ 横画面で2カラムレイアウト（左30%、右70%）になっている
- ✅ 配色が変更されている
- ✅ スマホでは縦並びのまま
- ✅ gap（余白）が適切に表示されている

---

## よくある質問

**Q: `git push` でエラーが出た場合は？**  
A: エラーメッセージを確認してください。認証エラーの場合は、Git Credential Managerで認証してください。

**Q: デプロイに失敗した場合は？**  
A: Vercelのダッシュボードで「Deployments」を確認し、エラーメッセージを確認してください。よくある原因はビルドエラーです。

**Q: 変更が反映されない場合は？**  
A: ブラウザのキャッシュをクリア（Ctrl + Shift + R）して、再度確認してください。
